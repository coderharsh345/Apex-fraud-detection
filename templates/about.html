    • Website Details:

    The Main Purpose of the site is "Online transactions involves the transfer and sharing of personal information (financial information, passwords, etc), and the misuse of this information leads to a host of frauds and scams, credit card fraud being one of the major threats. This can further lead into crimes like identity theft by the third person. One of the best ways to keep your business safe from cybercriminals is layer of security in payment. One such security layer is your payment gateway, which makes sure that there are no vulnerabilities present in the processing payments. This website is then used to recognize whether the transaction is fraudulent or a real transaction done by the owner. Our objective to make this website is to detect the fraudulent transactions while minimizing the fraud transaction. In Apex Credit Card Fraud Detection, we focused on pre-processing data sets and then analysing as well as on deploying multiple anomaly detection algorithms such as k-nearest neighbours and Logistic Regression Algorithm. Website structure do modelling past credit card transactions with the data of the ones that turned out to be fraud from the stored data set through machine learning."

    Apex fraud detection is of utmost importance, for any bank or financial organization . We've to see for all the potential fraud in order that consumers don't suffer to pay bill for goods that they haven't purchased or become a victim of card frauds in the other method. The survey conducted by Clifton Phua and his associates has revealed that techniques such as data processing applications, automated fraud detection, adversarial detection have given an unexpected results in success in some areas but they did not provide a permanent and consistent solution in finding fraud detection for longer period of time. Similar research was presented by Wen-Fang YU and Na Wang where they used algorithm like Outlier mining, Outlier detection mining and Distance sum algorithms to predict fraudulent transaction accurately in cases of emulation experiment of Mastercard transaction data set of one certain full service bank. Outlier is a data object that significantly deviates from the rest of the data objects and behaves in a different manner. It deals with detecting objects that are detached from the most system, the transactions that are fraud and not genuine. The aim here is, therefore, to create an internet site that checks whether a requested transaction may be a fraud or the real. Unconventional techniques such as hybrid datamining network classification algorithmizable to perceive illegal instances in an actual card transaction data set, supported network reconstruction algorithm that permits creating representations of the one instance deviation from a reference group have proved efficient typically on medium sized online transaction. In this machine learning project, we solve the problem of detecting credit card fraud transactions using python libraries such as Scikit Learn, Panda, MatplotLib, Pyplot, Graph, Seaborn, sklearn.linearmodel, Sklearn.matrics, sklearn.modelselection, NumPy, and few other python libraries. We solve the problem by creating a binary classifier and experimenting with various machine learning techniques and algorithm to classify the potential fraud transaction. Function used in the machine learning code: head(), tail(), info(), isnull().num(), value_counts(), .shape, .describe(), .mean(), .groupby(), .drop(), .fit() The dataset is used from the Kaggle including the data from year 2018-19. The dataset consists of 31 parameters columns. “Time" and “Amount” are the only aspects that were not modified with PCA. Total number of 284,807 transactions used with only 492 of them being fraud.
